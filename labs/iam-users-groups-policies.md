# IAM Users, Groups, and Policies Lab

## Goal

Practice the basics of AWS Identity and Access Management.

## Services

- IAM users
- IAM groups
- IAM policies
- Principle of least privilege

## Planned Steps

1. Review IAM concepts.
2. Create a test user if needed.
3. Create or inspect a group.
4. Attach a limited policy.
5. Review permissions.
6. Clean up lab resources.

## Security Notes

- Do not create long-term access keys unless needed.
- Do not give administrator access for simple labs.
- Use least privilege.
- Delete test users and credentials after practice.
